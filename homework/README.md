# Домашнее задание №1: Реализация алгоритма поиска с опечатками

## Описание задания
Реализовать консольное приложение на языке C# для вычисления расстояния Левенштейна между двумя строками.

## Требования
- Консольное приложение на C# с операторами верхнего уровня
- Пользователь вводит две строки, программа выводит расстояние Левенштейна
- Вечный цикл с выходом по команде "exit"
- Предоставить отчет с текстом программы и результатами выполнения

## Ответы на контрольные вопросы

### 1. Что такое расстояние Левенштейна?
Расстоянием Левенштейна называется минимальное количество действий, необходимых для преобразования одного слова в другое.
Основные операции, учитываемые при вычислении расстояния Левенштейна:

D (delete) — удаление символа

I (insert) — добавление символа

R (replace) — замена символа

M (match) — совпадение символов (не требует действий)
### 2. Чем отличается расстояние Левенштейна от расстояния Дамерау-Левенштейна?
Расстояние Дамерау-Левенштейна — это усовершенствованная версия расстояния Левенштейна, которая дополнительно учитывает операцию транспозиции (перестановки двух соседних символов) как единичное действие.

При использовании расстояния Дамерау-Левенштейна за единичное расстояние принимаются следующие действия:

I (insert) — добавление символа

D (delete) — удаление символа

R (replace) — замена символа

T (transposition) — перестановка двух соседних символов

### 3. Объясните алгоритм Вагнера-Фишера
Алгоритм Вагнера-Фишера использует динамическое программирование для вычисления расстояния Левенштейна.
Как работает:
Создается матрица размером (M+1) × (N+1), где M и N — длины строк.

Заполнение матрицы:

Первая строка: 0,1,2,...,N (вставки)

Первый столбец: 0,1,2,...,M (удаления)

Для каждой ячейки D[i,j] считаем минимум из трех вариантов:

D[i-1, j] + 1 — удаление

D[i, j-1] + 1 — вставка

D[i-1, j-1] + cost — замена (0 если символы совпадают, 1 если разные)

Дополнительно для Дамерау-Левенштейна:

Проверяем возможность транспозиции (перестановки соседних символов)

Если символы можно переставить, учитываем D[i-2, j-2] + cost

Результат — значение в правом нижнем углу матрицы.

Сложность: O(M × N) по времени и памяти.
